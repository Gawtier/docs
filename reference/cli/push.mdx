## Content outline

1. **Command Overview**
   ```bash
   forest push
   ```

2. **What It Does**
   - Uploads `.forestadmin-schema.json`
   - Syncs layout changes
   - Updates branch configuration
   - Validates changes

3. **When to Use**
   - After adding Smart Actions
   - After adding Smart Fields
   - After adding collections
   - After relationship changes
   - After any agent customization

4. **Push Process**
   - Read local schema
   - Compare with remote
   - Upload changes
   - Validate schema
   - Confirm success

5. **Options**
   - `--force` - Force push (skip confirmation)
   - `--dry-run` - Preview without pushing
   - `--branch <name>` - Push to specific branch

6. **Examples**
   ```bash
   # Standard push
   forest push

   # Preview changes
   forest push --dry-run

   # Force push
   forest push --force

   # Push to specific branch
   forest push --branch feature/new-field
   ```

7. **Validation**
   - Schema validation
   - Breaking change detection
   - Relationship checks
   - Type checking

8. **Errors & Warnings**
   - Invalid schema
   - Breaking changes
   - Merge conflicts
   - Network errors

9. **Best Practices**
   - Push frequently
   - Test locally first
   - Review changes
   - Use branches for features

10. **Troubleshooting**
    - Push rejected
    - Schema validation errors
    - Authentication issues
    - Network timeouts
