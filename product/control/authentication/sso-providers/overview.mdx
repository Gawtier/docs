## Description

Provider-specific guides for configuring Single Sign-On with Forest Admin.

## Supported Providers

### Major Identity Providers

- **[Azure AD / Entra ID](/product/control/authentication/sso-providers/azure)** - Microsoft's identity platform
- **[Okta](/product/control/authentication/sso-providers/okta)** - Leading enterprise identity provider
- **[Google Workspace](/product/control/authentication/sso-providers/google)** - Formerly G Suite
- **[AWS SSO / IAM Identity Center](/product/control/authentication/sso-providers/aws)** - Amazon's identity service

### Generic SAML

- **[Generic SAML 2.0](/product/control/authentication/sso-providers/generic-saml)** - For any SAML-compatible provider:
  - OneLogin
  - Auth0
  - Ping Identity
  - JumpCloud
  - Custom SAML IdP

## Choosing a Provider

If your organization already uses an identity provider, use that one. If you're choosing a provider:

- **Azure AD** - Best for Microsoft 365 organizations
- **Okta** - Best for enterprise SSO with many integrations
- **Google Workspace** - Best for Google-centric organizations
- **AWS SSO** - Best for AWS-heavy infrastructure

## Setup Process Overview

All providers follow a similar process:

1. **Forest Admin Configuration**
   - Enable SSO in project settings
   - Note the ACS URL and Entity ID

2. **IdP Configuration**
   - Create SAML application
   - Configure SAML settings
   - Set attribute mappings
   - Assign users/groups

3. **Metadata Exchange**
   - Download IdP metadata
   - Upload to Forest Admin
   - OR manually configure endpoints

4. **Testing**
   - Test with a user
   - Verify attribute mapping
   - Test login flow

5. **Enforcement** (optional)
   - Enable SSO-only login
   - Disable password authentication
