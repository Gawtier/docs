## Accessing Organization Settings

To configure SSO or SCIM:

1. Click your **Gravatar** in the bottom-left corner
2. Select **Organization Settings**
3. Navigate to the **SSO** or **SCIM** tab

## Prerequisites

<Warning>
This feature may not be available on all plans. Check the [pricing page](https://www.forestadmin.com/pricing) for more information, or [contact us](https://www.forestadmin.com/contact) to enable it for your organization.
</Warning>

## Configuration Steps

### Step 1: Create Application

In your Okta admin dashboard, click Create a new app integration and select SAML 2.0 and follow the wizard.

### Step 2: Obtain Metadata

Navigate to the Sign On tab in your newly created Okta application, locate the Metadata details section, and copy the Metadata URL.

### Step 3: Upload to Forest Admin

In Forest Admin, choose the "XML file upload or XML file endpoint" option and paste your Metadata URL into the designated field.

### Step 4: Testing

You can perform some tests before enabling it for your whole company.

## Required Configuration Settings

| Setting | Value |
|---------|-------|
| ACS URL | `https://api.forestadmin.com/api/saml/callback` |
| Audience URI (EntityID) | `forestadmin-OrganizationName` |
| Name ID Format | **EmailAddress** |
| Application Username | Email or None |
| Update Application Username | Create and update |

<Info>
Check the "Use this for Recipient URL and Destination URL" option for the ACS URL.
</Info>

## Optional Setting

**Default RelayState**: `{"organizationName": "<OrganizationName>", "destinationUrl": "organization.projects"}` (only needed for IDP-initiated login)

## Troubleshooting

- Verify all endpoints and certificate expiration dates
- Ensure Name ID format matches the email address used on Forest Admin accounts
